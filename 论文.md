# 基于Spring Boot的私人健身房管理系统的设计与实现

## 摘要
随着健身行业的精细化发展，私人健身房对会员管理、私教预约、器材维护等业务的数字化需求日益迫切。针对传统私人健身房管理模式效率低、数据统计不及时、业务流程不规范等问题，本文设计并实现了一套基于Spring Boot + Vue3的私人健身房管理系统。系统采用前后端分离的B/S架构，后端以Spring Boot为核心框架，结合MyBatis-Plus、MySQL、Redis、JWT等技术实现数据持久化、缓存优化与身份认证；前端基于Vue3和Element Plus构建交互界面。系统涵盖管理员端、会员端、私教端三大模块，实现了会员管理、私教预约、课程排期、器材库存、财务统计、权限管理等核心功能。测试结果表明，该系统能够满足私人健身房的日常管理需求，提升管理效率与用户体验，具有良好的实用性和可扩展性。

**关键词**：私人健身房；Spring Boot；Vue3；前后端分离；管理系统；JWT；Redis

## Abstract
With the refined development of the fitness industry, private gyms have an increasingly urgent demand for digitalization of businesses such as member management, personal trainer reservation, and equipment maintenance. Aiming at the problems of low efficiency, untimely data statistics, and irregular business processes in the traditional private gym management model, this paper designs and implements a private gym management system based on Spring Boot + Vue3. The system adopts a B/S architecture with separate front and back ends. The back end uses Spring Boot as the core framework, combined with MyBatis-Plus, MySQL, Redis, JWT and other technologies to achieve data persistence, cache optimization and identity authentication; the front end builds an interactive interface based on Vue3 and Element Plus. The system covers three major modules: administrator terminal, member terminal, and personal trainer terminal, realizing core functions such as member management, personal trainer reservation, course scheduling, equipment inventory, financial statistics, and permission management. Test results show that the system can meet the daily management needs of private gyms, improve management efficiency and user experience, and has good practicability and scalability.

**Keywords**：Private Gym; Spring Boot; Vue3; Separation of Front and Back Ends; Management System; JWT; Redis

## 第一章 绪论
### 1.1 研究背景与意义
#### 1.1.1 研究背景
近年来，居民健康意识的提升推动了健身行业的快速发展，私人健身房凭借“一对一私教”“个性化课程”等差异化优势成为健身市场的重要细分领域。与传统大型健身房相比，私人健身房的核心竞争力在于精细化的会员服务和高效的私教管理，但目前多数私人健身房仍采用“纸质记录+Excel统计”的管理模式，存在以下痛点：
- 会员信息分散，无法快速查询会员的消费记录、剩余课时、预约情况；
- 私教排期手动记录，易出现时间冲突，预约效率低；
- 器材维护、库存管理无系统化跟踪，易导致器材闲置或损坏；
- 财务数据统计依赖人工核算，耗时且易出错，无法实时掌握经营状况。

随着计算机技术和Web开发框架的成熟，基于Spring Boot、Vue等技术构建轻量化、高可用的管理系统，成为解决私人健身房管理痛点的有效途径。

#### 1.1.2 研究意义
**理论意义**：本文以私人健身房这一细分场景为研究对象，完善了中小型服务业管理系统的设计与实现思路，验证了Spring Boot + Vue前后端分离架构在小型管理系统中的适用性，为同类中小型管理系统的开发提供了理论参考。
**实践意义**：系统落地后可实现私人健身房业务的数字化管理，具体价值体现在：
- 提升管理效率：自动化处理会员预约、课程排期、财务统计等业务，减少人工操作；
- 优化用户体验：会员可在线预约私教、查询消费记录，私教可实时查看预约信息；
- 数据驱动决策：通过财务统计、会员行为分析等功能，为健身房运营提供数据支撑；
- 降低管理成本：减少纸质单据和人工核算的成本，降低管理失误率。

### 1.2 国内外研究现状
#### 1.2.1 国外研究现状
欧美国家健身行业数字化程度较高，如Mindbody、Glofox等健身管理系统已实现预约、支付、会员管理的全流程数字化，支持多终端适配和大数据分析，但此类系统多面向大型连锁健身房，功能冗余且成本较高，难以适配小型私人健身房的需求。

#### 1.2.2 国内研究现状
国内健身管理系统的研究多聚焦于大型健身房的综合管理，如涵盖门禁、收银、团课管理的一体化系统，但针对私人健身房“私教为主、会员量少、个性化强”的特点设计的系统较少，且部分系统存在界面复杂、操作繁琐、定制化难度大等问题。

## 第二章 相关技术与环境概述
### 2.1 后端技术栈
#### 2.1.1 Spring Boot框架
Spring Boot是由Pivotal团队开发的轻量级Java开发框架，基于Spring框架简化了应用的配置和部署流程。其核心优势包括：
- 自动配置：根据依赖自动配置项目，无需手动编写大量XML配置；
- 起步依赖：将常用依赖打包成starter，降低依赖管理复杂度；
- 嵌入式容器：内置Tomcat、Jetty等容器，支持一键启动；
- 监控与调试：提供Actuator组件，可实时监控应用运行状态。
本系统采用Spring Boot 2.7.x版本作为后端核心框架，实现业务逻辑的快速开发。

#### 2.1.2 MyBatis-Plus框架
MyBatis-Plus（简称MP）是基于MyBatis的增强型持久层框架，在MyBatis的基础上扩展了CRUD操作、分页查询、条件构造器等功能，无需编写大量XML映射文件即可实现数据库操作，大幅提升开发效率。本系统使用MP实现数据库的增删改查，简化持久层代码开发。

#### 2.1.3 MySQL数据库
MySQL是一款开源的关系型数据库管理系统，具有轻量级、高性能、易部署的特点，支持事务、索引、视图等核心功能，适用于中小型应用的数据库存储。本系统采用MySQL 8.0版本存储会员、私教、课程、订单等核心数据，并通过索引优化查询性能。

#### 2.1.4 Redis缓存技术
Redis是一款高性能的键值对缓存数据库，支持字符串、哈希、列表、集合等多种数据结构，可用于缓存热点数据、减轻数据库压力。本系统将财务统计结果、会员常用信息等热点数据缓存至Redis，提升系统响应速度。

#### 2.1.5 JWT身份认证
JWT（JSON Web Token）是一种轻量级的身份认证令牌，由头部、载荷、签名三部分组成，可在前后端之间安全传递用户身份信息。本系统采用JWT实现无状态的身份认证，用户登录后生成JWT令牌，后续请求通过令牌验证身份，避免频繁查询数据库。

### 2.2 前端技术栈
#### 2.2.1 Vue3框架
Vue3是一款渐进式JavaScript前端框架，采用组合式API（Composition API）提升代码的复用性和可维护性，支持虚拟DOM、响应式数据绑定等核心特性，相比Vue2具有更高的性能和更好的类型支持。本系统基于Vue3构建前端界面，实现组件化开发。

#### 2.2.2 Element Plus组件库
Element Plus是基于Vue3的桌面端组件库，提供了表单、表格、弹窗、导航等常用组件，支持自定义样式和国际化，可快速构建美观、易用的前端界面。本系统使用Element Plus实现核心功能的界面展示，减少前端开发工作量。

#### 2.2.3 Axios网络请求
Axios是一款基于Promise的HTTP客户端，支持请求拦截、响应拦截、请求取消、错误处理等功能，可在浏览器和Node.js环境中使用。本系统通过Axios封装前端请求，实现与后端接口的交互，并在拦截器中统一处理JWT令牌的携带和过期逻辑。

## 第三章 系统需求分析
### 3.1 可行性分析
#### 3.1.1 技术可行性
本系统采用的Spring Boot、Vue3、MySQL等技术均为成熟的开源技术，相关文档和社区资源丰富，开发门槛低；开发团队具备Java和前端开发经验，能够掌握核心技术的使用；系统采用前后端分离架构，可分模块开发、测试和部署，降低技术实现难度。因此，从技术层面来看，系统开发具备可行性。

#### 3.1.2 经济可行性
系统开发基于开源技术栈，无需支付商业软件授权费用；开发周期约3个月，人力成本可控；系统上线后可替代人工管理，降低健身房的运营成本（如减少1-2名管理人员），短期内可收回开发成本。因此，从经济层面来看，系统开发具备可行性。

#### 3.1.3 操作可行性
系统界面采用Element Plus组件库构建，遵循“简洁、易用”的设计原则，管理员、私教、会员均无需专业的计算机知识，通过简单培训即可掌握操作方法；系统支持PC端和移动端适配，可随时随地进行操作。因此，从操作层面来看，系统开发具备可行性。

### 3.3 非功能性需求分析
#### 3.3.1 性能需求
- 系统响应时间：普通查询请求≤1s，复杂统计请求≤3s；
- 并发处理能力：支持≥50个并发用户操作，无卡顿或数据错误；
- 数据存储能力：支持存储≥1000条会员记录、≥100条私教记录、≥10000条消费记录。

#### 3.3.2 安全性需求
- 身份认证：采用JWT令牌认证，令牌有效期2小时，过期需重新登录；
- 数据加密：用户密码采用BCrypt算法加密存储，敏感数据（如身份证号）脱敏展示；
- 权限控制：基于RBAC模型实现权限控制，不同角色仅能访问对应功能模块；
- 日志记录：记录关键操作（如账号登录、数据修改、财务操作），便于追溯。

#### 3.3.3 易用性需求
- 界面设计：遵循“少操作、多反馈”原则，核心功能操作步骤≤3步；
- 提示信息：操作成功/失败、输入错误等场景提供清晰的文字提示；
- 兼容性：支持Chrome、Firefox、Edge等主流浏览器，适配1366×768及以上分辨率。

#### 3.3.4 可扩展性需求
- 模块解耦：各功能模块独立开发、部署，便于后续新增功能（如在线支付、会员推荐）；
- 接口标准化：遵循RESTful API规范设计接口，便于对接第三方系统（如微信小程序）。

## 第四章 系统设计
### 4.1 系统总体架构设计
#### 4.1.1 架构整体设计
系统采用前后端分离的B/S架构，整体架构分为三层：
1. **表现层**：基于Vue3 + Element Plus构建的前端界面，负责用户交互和数据展示；
2. **应用层**：基于Spring Boot的后端服务，负责业务逻辑处理、接口提供、身份认证；
3. **数据层**：由MySQL和Redis组成，负责持久化数据存储和热点数据缓存。

前后端通过RESTful API交互，前端发送HTTP请求至后端接口，后端返回JSON格式数据，实现数据解耦和跨终端适配。

#### 4.1.2 后端分层架构
后端采用分层设计，各层职责清晰：
| 层级          | 职责                                                                 |
|---------------|----------------------------------------------------------------------|
| Controller层  | 接收前端请求，参数校验，调用Service层方法，返回响应结果               |
| Service层     | 实现核心业务逻辑，分为接口（IService）和实现类（ServiceImpl）         |
| Mapper层      | 基于MyBatis-Plus实现数据库操作，无需编写SQL即可完成CRUD               |
| Entity层      | 定义数据库表对应的实体类，映射表字段和属性                           |
| DTO层         | 数据传输对象，用于前后端数据交互，避免直接暴露实体类字段               |
| Util层        | 工具类，如JWT生成/验证、密码加密、日期处理等                         |
| Config层      | 配置类，如Redis配置、MyBatis-Plus配置、跨域配置等                     |

#### 4.1.3 前端架构设计
前端采用组件化和模块化设计，核心结构如下：
| 层级          | 职责                                                                 |
|---------------|----------------------------------------------------------------------|
| 页面层（views）| 对应系统的功能页面，如会员管理页、私教预约页等                       |
| 组件层（components）| 可复用的UI组件，如表格组件、表单组件、弹窗组件等                     |
| 请求层（api） | 封装Axios请求，统一管理后端接口地址，处理请求/响应拦截               |

### 4.2 数据库设计

3. 为常用查询字段（如会员ID、私教ID）建立索引，提升查询性能；
4. 字段命名采用下划线命名法，与实体类属性的驼峰命名法对应。

### 4.3 模块详细设计
#### 4.3.1 权限管理模块设计
权限管理模块基于RBAC（基于角色的访问控制）模型设计，核心流程：
1. 管理员为用户分配角色（如会员、私教）；
2. 角色关联菜单（功能模块），定义角色可访问的页面和接口；
3. 用户登录后，系统根据角色加载对应的菜单和权限；
4. 前端路由守卫拦截未授权的页面访问，后端接口拦截未授权的请求。

#### 4.3.2 会员管理模块设计
会员管理模块分为管理员端和会员端，核心逻辑：
- 管理员端：新增会员（关联用户表）、编辑会员信息、查询会员消费记录、统计会员剩余课时；
- 会员端：完善个人信息、查询消费记录、预约私教（扣减课时）、充值课时。

#### 4.3.3 私教预约模块设计
私教预约模块是系统核心模块，核心逻辑：
1. 会员选择私教和课程，选择预约时间（需避开私教已排期的时间）；
2. 系统校验会员剩余课时是否充足，充足则创建预约记录（状态为待确认）；
3. 私教查看预约记录，确认/拒绝预约（拒绝则释放预约时间）；
4. 预约时间结束后，私教确认授课完成，系统扣减会员课时，计入私教业绩。

#### 4.3.4 财务统计模块设计
财务统计模块基于订单表和私教业绩表设计，核心逻辑：
1. 每日/每月自动统计充值金额、课程消费金额、私教工资支出；
2. 将统计结果缓存至Redis，提升查询速度；
3. 支持导出财务报表（Excel格式）；
4. 管理员可查看财务明细，追溯每笔收支记录。

### 4.4 核心业务流程设计
#### 4.4.1 私教预约流程
1. 会员登录系统，进入私教列表页，查看私教的排期和课程信息；
2. 会员选择私教、课程和预约时间，提交预约请求；
3. 系统校验：
   - 预约时间是否与私教已排期时间冲突；
   - 会员剩余课时是否≥课程所需课时；
4. 校验通过：创建预约记录（状态：待确认），返回预约成功提示；
5. 私教登录系统，查看待确认预约列表，选择确认/拒绝；
   - 确认：预约状态改为“已确认”，锁定私教该时段；
   - 拒绝：预约状态改为“已取消”，释放私教该时段；
6. 预约时段结束后，私教点击“授课完成”，系统扣减会员剩余课时，预约状态改为“已完成”；
7. 会员可查看预约记录和课时扣减情况。

#### 4.4.2 会员充值消费流程
1. 会员进入充值页面，选择充值金额（对应课时数），提交充值请求；
2. 系统创建充值订单（状态：待支付），会员完成支付后，订单状态改为“已支付”；
3. 系统增加会员剩余课时，记录充值记录；
4. 会员预约私教并完成授课后，系统创建消费订单，扣减剩余课时，订单状态改为“已支付”；
5. 管理员可查看充值和消费订单的明细和统计数据。

### 4.5 接口设计（RESTful API规范）
系统接口遵循RESTful API规范，核心设计原则：
1. 使用HTTP方法表示操作类型：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）；
2. 接口路径采用名词复数形式，如`/api/members`（会员）、`/api/trainers`（私教）；
3. 接口版本号放入请求头或路径，如`/api/v1/members`；
4. 响应数据格式统一：

  "msg": "操作成功", // 提示信息

##### （1）JWT工具类
```java
package com.gym.util;

import io.jsonwebtoken.Claims;
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.stereotype.Component;

import java.util.Date;
import java.util.HashMap;
import java.util.Map;

/**
 * JWT工具类，用于生成和验证令牌
 */
@Component
public class JwtUtil {

    // JWT密钥
    @Value("${jwt.secret}")
    private String secret;

    // 令牌有效期（毫秒）
    @Value("${jwt.expire}")
    private long expire;

    /**
     * 生成令牌
     * @param userDetails 用户信息
     * @return JWT令牌
     */
    public String generateToken(UserDetails userDetails) {
        Map<String, Object> claims = new HashMap<>();
        // 设置令牌载荷，存储用户名
        claims.put("username", userDetails.getUsername());
        return Jwts.builder()
                .setClaims(claims)
                .setSubject(userDetails.getUsername())
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + expire))
                .signWith(SignatureAlgorithm.HS256, secret)
                .compact();
    }

    /**
     * 验证令牌有效性
     * @param token 令牌
     * @param userDetails 用户信息
     * @return 是否有效
     */
    public boolean validateToken(String token, UserDetails userDetails) {
        String username = extractUsername(token);
        return username.equals(userDetails.getUsername()) && !isTokenExpired(token);
    }

    /**
     * 从令牌中提取用户名
     * @param token 令牌
     * @return 用户名
     */
    public String extractUsername(String token) {
        return extractClaims(token).getSubject();
    }

    /**
     * 检查令牌是否过期
     * @param token 令牌
     * @return 是否过期
     */
    private boolean isTokenExpired(String token) {
        return extractClaims(token).getExpiration().before(new Date());
    }

    /**
     * 提取令牌载荷
     * @param token 令牌
     * @return 载荷
     */
    private Claims extractClaims(String token) {
        return Jwts.parser()
                .setSigningKey(secret)
                .parseClaimsJws(token)
                .getBody();
    }
}
```

    private AuthenticationManager authenticationManager;

    @Autowired
    private UserDetailsService userDetailsService;

    @Autowired
    private JwtUtil jwtUtil;

    @Autowired
    private SysUserService sysUserService;

        data.put("token", token);
        data.put("userId", sysUser.getId());
        data.put("roleId", sysUser.getRoleId());
        data.put("username", sysUser.getUsername());

package com.gym.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.gym.entity.GymMember;

import org.springframework.beans.factory.annotation.Autowired;

import java.time.LocalDateTime;

/**
 * 会员服务实现类
 */
@Service
public class GymMemberServiceImpl extends ServiceImpl<GymMemberMapper, GymMember> implements GymMemberService {

    @Autowired
    private SysUserService sysUserService;

    /**
     * 新增会员
     * @param gymMember 会员信息
     * @return 是否成功
     */
    @Override
    public boolean addMember(GymMember gymMember) {
        // 1. 校验用户是否存在
        SysUser sysUser = sysUserService.getById(gymMember.getUserId());
        if (sysUser == null) {
            throw new RuntimeException("关联用户不存在");
        }
        // 2. 设置注册时间和剩余课时（默认0）
        gymMember.setRegisterTime(LocalDateTime.now());
        gymMember.setRemainingHours(0);
        gymMember.setUpdateTime(LocalDateTime.now());
        // 3. 保存会员信息
        return save(gymMember);
    }

    /**
     * 充值课时
     * @param memberId 会员ID
     * @param hours 充值课时数
     * @return 是否成功
     */
    @Override
    public boolean rechargeHours(Long memberId, Integer hours) {
        if (hours <= 0) {
            throw new RuntimeException("充值课时数必须大于0");
        }
        GymMember gymMember = getById(memberId);
        if (gymMember == null) {
            throw new RuntimeException("会员不存在");
        }
        // 累加剩余课时
        gymMember.setRemainingHours(gymMember.getRemainingHours() + hours);
        gymMember.setUpdateTime(LocalDateTime.now());
        return updateById(gymMember);
    }

    /**
     * 扣减课时
     * @param memberId 会员ID
     * @param hours 扣减课时数
     * @return 是否成功
     */
    @Override
    public boolean deductHours(Long memberId, Integer hours) {
        GymMember gymMember = getById(memberId);
        if (gymMember == null) {
            throw new RuntimeException("会员不存在");
        }
        if (gymMember.getRemainingHours() < hours) {
            throw new RuntimeException("剩余课时不足");
        }
        // 扣减剩余课时
        gymMember.setRemainingHours(gymMember.getRemainingHours() - hours);
        gymMember.setUpdateTime(LocalDateTime.now());
        return updateById(gymMember);
    }

    /**
     * 根据用户ID查询会员信息
     * @param userId 用户ID
     * @return 会员信息
     */
    @Override
    public GymMember getByUserId(Long userId) {
        return getOne(new LambdaQueryWrapper<GymMember>().eq(GymMember::getUserId, userId));

package com.gym.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.gym.entity.GymMember;
import com.gym.entity.GymReservation;

import com.gym.service.GymMemberService;
import com.gym.service.GymReservationService;
import com.gym.service.GymTrainerService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;

/**
 * 预约服务实现类
 */
@Service
public class GymReservationServiceImpl extends ServiceImpl<GymReservationMapper, GymReservation> implements GymReservationService {

    @Autowired
    private GymMemberService gymMemberService;

    @Autowired
    private GymTrainerService gymTrainerService;

    /**
     * 会员预约私教
     * @param reservation 预约信息
     * @return 是否成功
     */
    @Override
    public boolean reserveTrainer(GymReservation reservation) {
        // 1. 校验会员和私教是否存在
        GymMember member = gymMemberService.getById(reservation.getMemberId());
        GymTrainer trainer = gymTrainerService.getById(reservation.getTrainerId());
        if (member == null) {
            throw new RuntimeException("会员不存在");
        }
        if (trainer == null) {
            throw new RuntimeException("私教不存在");
        }

        // 2. 校验预约时间是否冲突
        LambdaQueryWrapper<GymReservation> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(GymReservation::getTrainerId, reservation.getTrainerId())
                .eq(GymReservation::getReserveTime, reservation.getReserveTime())
                .in(GymReservation::getStatus, 0, 1); // 待确认、已确认的预约
        if (count(wrapper) > 0) {
            throw new RuntimeException("该时段已被预约，请选择其他时间");
        }

        // 3. 校验会员剩余课时
        Integer courseHours = 1; // 假设每节课程1课时
        if (member.getRemainingHours() < courseHours) {
            throw new RuntimeException("剩余课时不足，请充值");
        }

        // 4. 创建预约记录
        reservation.setStatus(0); // 待确认
        reservation.setCreateTime(LocalDateTime.now());
        reservation.setUpdateTime(LocalDateTime.now());
        return save(reservation);
    }

    /**
     * 私教确认预约
     * @param reservationId 预约ID
     * @return 是否成功
     */
    @Override
    public boolean confirmReservation(Long reservationId) {
        GymReservation reservation = getById(reservationId);
        if (reservation == null) {
            throw new RuntimeException("预约记录不存在");
        }
        if (reservation.getStatus() != 0) {
            throw new RuntimeException("该预约已处理，无需重复确认");
        }
        reservation.setStatus(1); // 已确认
        reservation.setUpdateTime(LocalDateTime.now());
        return updateById(reservation);
    }

    /**
     * 完成预约（扣减课时）
     * @param reservationId 预约ID
     * @return 是否成功
     */
    @Override
    public boolean completeReservation(Long reservationId) {
        GymReservation reservation = getById(reservationId);
        if (reservation == null) {
            throw new RuntimeException("预约记录不存在");
        }
        if (reservation.getStatus() != 1) {
            throw new RuntimeException("该预约未确认，无法完成");
        }
        // 扣减会员课时
        gymMemberService.deductHours(reservation.getMemberId(), 1);
        // 更新预约状态
        reservation.setStatus(3); // 已完成
        reservation.setUpdateTime(LocalDateTime.now());
        return updateById(reservation);
    }
}

package com.gym.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.gym.entity.GymOrder;
import com.gym.mapper.GymOrderMapper;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;

import java.math.BigDecimal;
import java.time.LocalDate;
import java.time.LocalDateTime;
import java.time.LocalTime;
import java.util.concurrent.TimeUnit;

/**
 * 订单（财务）服务实现类
 */
@Service
public class GymOrderServiceImpl extends ServiceImpl<GymOrderMapper, GymOrder> implements GymOrderService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    /**
     * 统计今日收入
     * @return 今日收入
     */
    @Override
    public BigDecimal getTodayIncome() {
        String redisKey = "gym:statistics:today_income:" + LocalDate.now();
        // 先从Redis获取
        BigDecimal todayIncome = (BigDecimal) redisTemplate.opsForValue().get(redisKey);
        if (todayIncome != null) {
            return todayIncome;
        }
        // Redis无数据，从数据库查询
        LocalDateTime startTime = LocalDateTime.of(LocalDate.now(), LocalTime.MIN);
        LocalDateTime endTime = LocalDateTime.of(LocalDate.now(), LocalTime.MAX);
        LambdaQueryWrapper<GymOrder> wrapper = new LambdaQueryWrapper<>();
        wrapper.ge(GymOrder::getCreateTime, startTime)
                .le(GymOrder::getCreateTime, endTime)
                .eq(GymOrder::getOrderStatus, 1) // 已支付
                .eq(GymOrder::getOrderType, 1); // 课程消费
        todayIncome = baseMapper.sumAmount(wrapper);
        if (todayIncome == null) {
            todayIncome = BigDecimal.ZERO;
        }
        // 存入Redis，缓存1小时
        redisTemplate.opsForValue().set(redisKey, todayIncome, 1, TimeUnit.HOURS);
        return todayIncome;
    }
}

## 第七章 总结与展望
### 7.1 系统总结
本文针对私人健身房管理的痛点，设计并实现了基于Spring Boot + Vue3的私人健身房管理系统。系统采用前后端分离的B/S架构，后端基于Spring Boot、MyBatis-Plus、MySQL、Redis、JWT等技术实现业务逻辑和数据管理，前端基于Vue3、Element Plus构建交互界面。系统涵盖管理员端、会员端、私教端三大模块，实现了会员管理、私教预约、课程排期、器材管理、财务统计等核心功能。

测试结果表明，系统能够有效解决传统私人健身房管理效率低、数据统计不及时等问题，提升了健身房的管理效率和用户体验。系统的主要优势体现在：
1. 前后端分离架构，便于模块扩展和维护；
2. Redis缓存热点数据，提升系统响应速度；
3. 基于JWT的身份认证，保证系统安全性；
4. 界面简洁易用，降低操作门槛。

### 7.2 系统不足
尽管系统满足了核心需求，但仍存在以下不足：
1. 缺乏移动端适配：目前仅支持PC端访问，会员和私教无法通过手机便捷操作；
2. 无在线支付功能：会员充值需线下完成，无法通过微信/支付宝在线支付；
3. 数据可视化程度低：财务统计仅展示数字，缺乏图表化展示；
4. 无智能推荐功能：无法根据会员的健身记录推荐私教或课程。

### 7.3 未来展望
针对系统的不足，未来可从以下方向优化：
1. 移动端开发：基于微信小程序或uniapp开发移动端版本，支持会员和私教的移动操作；
2. 集成在线支付：对接微信支付、支付宝等支付接口，实现会员充值的在线支付；
